## Честное использование

API предоставляется игрокам для упрощения ставок и не может использоваться для сбора данных, скрапинга или любых других целей. Использование API должно быть пропорционально ставочным действиям, как это определяется в каждом конкретном случае компанией Pinc000.

Если не было явно согласовано в письменной форме с Pinc000, коммерческое использование API приведет к постоянной приостановке вашего доступа.

Вы не будете пытаться или поощрять других: 

- Вмешиваться в работу API, нарушать его функционирование или отключать его функции;
- Использовать API в коммерческих целях без письменного согласия с Pinc000;
- Продавать, сдавать в аренду, предоставлять в лизинг, сублицензировать, перераспределять или синдицировать API третьим лицам без предварительного письменного разрешения от Pinc000.

### Правила 

1\. Вызовы Delta и Snapshot поддерживаются в эндпоинтах `/fixtures`, `/fixtures/special`, `/odds` и `/odds/special`. Вызовы Delta возвращают изменения с момента указанного значения `since`. Для вызовов Delta параметр `since` не должен быть равен 0 или 1, он всегда должен быть установлен в значение последнего свойства из ответа предыдущего вызова. Вызовы Snapshot возвращают текущее состояние, параметр `since` не должен быть предоставлен для вызовов Snapshot.

2\. Всегда сначала выполняйте вызов Snapshot, а затем продолжайте с вызовами Delta. Это приведет к более быстрому времени отклика и меньшему объему данных в ответе. В результате клиент получит обновления odds/fixtures быстрее.

3\. Клиент не должен вызывать эндпоинты `/odds` или `/fixtures` для каждой спортивной лиги или события в цикле. Если клиент заинтересован только в определенных лигах, параметр `leagueIds` должен быть установлен со всеми идентификаторами лиг. То же самое касается параметра `eventIds`: клиент должен использовать его только в случае интереса к конкретным событиям, в этом случае все идентификаторы событий должны быть указаны в одном вызове.

4\. Следующие ограничения должны соблюдаться для вызова `/sports`: 

- Запросы к `/sports` должны быть ограничены до одного раза в 60 секунд. Список видов спорта не изменяется часто. Подсчет активных событий является устаревшей функциональностью, которая в конечном итоге будет отключена.

5\. Следующие ограничения должны соблюдаться для каждого спорта: 

- Вызов Snapshot к эндпоинтам `/fixtures` и `/odds` должен быть ограничен до одного раза каждые 60 секунд, независимо от параметров `leagueIds`, `eventIds` или `islive`.

- Вызовы Delta к эндпоинтам `/fixtures` и `/odds` должны быть ограничены до одного раза каждые 5 секунд, независимо от параметров `leagueIds`, `eventIds` или `islive`.       

- Вызовы к `/leagues` должны быть ограничены до одного раза каждые 60 секунд.

6\. Клиент не должен вызывать эндпоинт `/line` в цикле. Цель этого эндпоинта — проверка цены перед размещением ставки

### Следующие ограничения должны соблюдаться для каждого спорта:

- Запросы к операциям /fixtures и /odds без параметра since должны быть ограничены до одного раза каждые 60 секунд;
- Запросы к операциям /fixtures и /odds с параметром since должны быть ограничены до одного раза каждые 5 секунд.
